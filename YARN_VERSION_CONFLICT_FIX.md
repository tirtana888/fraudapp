# Yarn Version Conflict Fix - Buildpack Compatibility

## Problem
Firebase deployment failing with "Unknown Syntax Error" on `yarn install` command. Buildpack was using old flags incompatible with Yarn v4 detection.

## Root Cause
- Deployment system detected Yarn but used incompatible flags
- Potential conflict between Yarn Classic (v1) and Yarn Modern (v2+)
- Lock file format or configuration causing buildpack confusion

## Solution Implemented

### 1. Verified Package Manager Configuration

**Root Directory (`/app/package.json`):**
```json
{
  "packageManager": "yarn@1.22.22",
  "engines": {
    "node": ">=18.0.0",
    "yarn": ">=1.22.0"
  }
}
```

**Functions Directory (`/app/functions/package.json`):**
```json
{
  "packageManager": "yarn@1.22.22",
  "engines": {
    "node": ">=18",
    "yarn": ">=1.22.0"
  }
}
```

### 2. Complete Clean & Reinstall

**Root Directory:**
```bash
cd /app
# Remove ALL lock files and dependencies
rm -rf node_modules yarn.lock package-lock.json .yarn .pnp.* yarn-error.log

# Clean Yarn cache
yarn cache clean

# Fresh install with Yarn Classic
yarn install
# Result: ✅ yarn.lock (152KB) - Yarn Classic v1 format
```

**Functions Directory:**
```bash
cd /app/functions
# Remove ALL lock files and dependencies
rm -rf node_modules yarn.lock package-lock.json .yarn .pnp.* yarn-error.log

# Fresh install with Yarn Classic
yarn install
# Result: ✅ yarn.lock (97KB) - Yarn Classic v1 format
```

### 3. Enhanced Yarn Configuration

**Updated `/app/.yarnrc`:**
```yaml
# Yarn configuration
# Force Yarn Classic v1 (NOT Yarn v2+)
# This prevents buildpack from using incompatible Yarn v4

# Ensure integrity check
--check-files true

# Disable PnP (Yarn v2+ feature)
--disable-pnp true
```

**Updated `/app/functions/.yarnrc`:**
```yaml
# Yarn configuration for functions
# Force Yarn Classic v1 (NOT Yarn v2+)

# Ensure integrity check
--check-files true

# Disable PnP (Yarn v2+ feature)
--disable-pnp true
```

## Troubleshooting

### If Deployment Still Fails

1. **Verify Yarn Version in Build Logs:**
   ```
   Look for: "yarn install v1.22.22"
   NOT: "yarn install v4.x.x"
   ```

2. **Check Lock File Format:**
   ```bash
   head -5 yarn.lock
   # Must show: "# yarn lockfile v1"
   ```

3. **Verify No Yarn v2+ Files:**
   ```bash
   find . -name ".yarnrc.yml" -o -name ".pnp.*"
   # Should return nothing
   ```

4. **Test Locally:**
   ```bash
   rm -rf node_modules
   yarn install --frozen-lockfile
   # Should complete without errors
   ```
